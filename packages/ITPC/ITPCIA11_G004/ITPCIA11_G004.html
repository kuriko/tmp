コマンドラインとは？

コマンドライン は、コンピューターに対して コマンド と呼ばれる命令をキーボードから入力することで、コンピューターを操作する画面です。

// TODO: コマンドラインの画像

コマンドラインは、コンソール や ターミナル と呼ばれることもあります。

コマンドラインを使うために、

  • Windows では「PowerShell」や「コマンドプロンプト」や「Windows Terminal」
  • Mac では「ターミナル」や「iTerm2」

というソフトウェアを使います。

GUI と CLI

現在のパソコンの一般的な操作は、マウスやトラックパッドを使った視覚的な操作です。これを GUI（Graphical User Interface グラフィカル・ユーザー・インターフェース）といいます。

一方、コマンドラインを使用して文字の入力だけで操作することを、CLI（Command Line Interface コマンドライン・インターフェース）または CUI（Character User Interface キャラクター・ユーザー・インターフェース）といいます。

豆知識 :   用語の由来

先ほど述べたとおり、コマンドラインはコンソールやターミナルと呼ばれることもあります。
参考までに、用語の由来を紹介しますが、覚えなくて構いません。


  • コマンドライン ... コマンド（命令）を入力する「行」という意味が由来です。コマンドの行をいくつも重ねていくことで、様々な操作を行います。
  • コンソール ... 「制御装置」や「制御卓」、「操作盤」という意味です。かつての大型コンピューターでは、コンピューターを操作するための卓上の装置が別に用意されていました。これの名残りで、今でもコンピューターの操作画面のことをコンソールと読んでいるのです。
  • ターミナル ... ターミナル（terminal）という英単語は「終端・端末」という意味です。昔の大型コンピューターはコンピューター本体と制御装置が分かれていて、ケーブルの終端に制御装置があったことから「ターミナル」（日本語では「端末」）と呼ばれています。

現在では、どれも同じ意味だと考えて実用上問題ありません。

コマンドラインでのファイル操作

注意: これより後の解説は 4 章 基本操作をしてみよう（Mac 編）- ファイルやフォルダ操作の基本を学ぼう の内容を前提としています。

これまでは、Finder というソフトウェアを使ってファイルやフォルダの操作を行ってきたと思います。
ここでは、コマンドライン（ターミナル）上でのファイル・フォルダ操作の方法を学びます。

ディレクトリと階層構造

コマンドラインでは、フォルダのことを ディレクトリ といいます。
フォルダとディレクトリは同じものだと考えて、実用上問題ありません。
以下では、フォルダをディレクトリと呼びます。

ディレクトリの中には、ファイルや別のディレクトリを入れることができます。

したがって、ディレクトリの中にディレクトリがあって、さらにその中にディレクトリがある、というような入れ子の構造をとることができます。
このような構造を、ディレクトリの 階層構造 といいます。

ルートディレクトリ

階層構造の一番上にあるディレクトリを ルートディレクトリ といいます。
ルートディレクトリは階層構造の一番上にあるので、全てのディレクトリはルートディレクトリの中（ルートディレクトリの下）にあります。

カレントディレクトリ

Finder とコマンドラインの大きな違いは、「ユーザーが今どのディレクトリにいるのか」 ということを気にするかどうかです。

コマンドラインでは、「ユーザーが今どのディレクトリにいるのか」が非常に重要です。

ユーザーが今いるディレクトリのことを カレントディレクトリ（Current Directory 現在のディレクトリ）といいます。

コマンドラインを使う上では、カレントディレクトリがどこなのかを明確に意識する必要があります。
カレントディレクトリがどこなのかによって、コマンドの結果は全く異なることがあるからです。

パス

あるファイルやディレクトリが階層構造のどこに位置するのかを文字列で示したものを ファイルパス や ディレクトリパス あるいは単に パス といいます。
パス（path）は「小道」を意味する英単語で、そのファイルやディレクトリにたどり着くための道順をイメージするとよいでしょう。

Finder でファイルやディレクトリのパスを見てみよう

「Finder」を開いたら、ツールバーの「表示」から「パスバーを表示」を押しましょう。
もし、「パスバーを非表示」となっていたら、何も押さなくて構いません。

適当なファイルまたはフォルダをマウス・トラックパッドで選択してください。
ここでは、書類フォルダ内に homework.txt というファイルを作って選択してみました。

ファイルの拡張子

ここで、homework.txt というファイルの .txt の部分は、ファイルの 拡張子 です。
拡張子はファイルの種類を区別するためにつける .（ドット）以下の文字列です。
.txt は、そのファイルが「テキストファイル」であることを表しています。

Finder のメニューから「環境設定」>「詳細」>「全てのファイル名拡張子を表示」にチェックを入れることで、拡張子を表示させることができます。
なお、コマンドラインを使うときは、デフォルトで拡張子が表示されます。

さて、ファイルを選択すると Finder のウィンドウの下部に

Macintosh HD > ユーザ > あなたのユーザー名 > 書類 > homework.txt

などのように表示されるのではないでしょうか。これが homework.txt というファイルの位置を示すパスです。

絶対パスと相対パス

ルートディレクトリ（ / ）を起点として、ファイルやディレクトリの位置を示す方法を 絶対パス といいます。例えば /Users/あなたのユーザー名/Documents/homework.txt は絶対パスです。

一方、カレントディレクトリ（ . ）を起点として、ファイルやディレクトリの位置を示す方法を 相対パス といいます。カレントディレクトリが Documents だとすると、上記のファイルは相対パスで ./homework.txt と表されます。

ファイル操作のコマンド

Mac のターミナルでファイル操作を行うためのコマンドを紹介していきます。

多いですが、必要な時に調べながら使っていくことができれば十分です。
例えば、「mac コマンド 削除」などと検索サイトで検索することで、必要な rm コマンドを見つけることができるでしょう。

コマンド できる操作
pwd カレントディレクトリの表示
ls ファイル・ディレクトリの一覧の表示
cd ディレクトリの移動
mkdir ディレクトリの作成
rm ファイルやディレクトリの削除

それでは、1 つずつ使い方を見ていきましょう。

pwd コマンド

pwd は、Print Working Directory (プリント・ワーキング・ディレクトリ）の略で、「作業中のディレクトリを表示せよ」という意味です。
このコマンドが実行された時点での、カレントディレクトリを表示します。

pwd のイメージ

ターミナルに


pwd

と入力し Enter キーを押します。

Users/あなたのユーザー名

と表示されるはずです。

これが pwd コマンドによって表示された現在のディレクトリ（カレントディレクトリ）です。
これは、ルートディレクトリから見て Users というディレクトリ内の あなたのユーザー名 というディレクトリがカレントディレクトリであることを示しています。
つまり、あなたは今、Users/あなたのユーザー名 というディレクトリにいることを意味します。

今どこにいるのか（カレントディレクトリ）を知りたいとき、pwd コマンドを使用しましょう。

ls コマンド

ls （エル・エス）は LiSt の略で、「（ファイルやディレクトリの）リストを表示せよ」という意味のコマンドです。
カレントディレクトリの中にあるファイルとディレクトリの一覧 (リスト) を表示します。

ls のイメージ 

ls

と入力して Enter キーを押します。
l はアルファベット小文字の「エル」です。 I（アルファベット大文字のアイ）や 数字の 1 と間違えないように注意してください。

Applications    Downloads   Music
Desktop         Library     Pictures
Documents       Movies      Public

のように表示されたのではないでしょうか（細かい部分が違っても、気にする必要はありません）。

また、ls コマンドに -a オプションを付けて


ls -a

を実行すると、. から始まるディレクトリやファイル（ドットファイルといいます）も含めて表示できます。ls と -a の間は半角スペースを開けます。

カレントディレクトリの中にあるファイルやディレクトリの一覧を見たいとき、pwd コマンドを使用しましょう。

cd コマンド

cd は、Change Directory（チェンジ・ディレクトリ）の略で、「ディレクトリを変更せよ」という意味のコマンドです。
カレントディレクトリから別のディレクトリに移動します。cd の後ろにスペースをあけて、移動先のディレクトリを記述します。


cd のイメージ 

cd ..
pwd

と入力してみましょう。1 行ずつ Enter キーを押すことに注意してください。

/Users

と表示されたのではないでしょうか。
.. は 1 つ上の階層のディレクトリを表すので、これまでいた /Users/あなたのユーザー名 の 1 つ上のディレクトリ /Users に移動したのです。

また、この時点で


ls

と入力すると、

Shared  あなたのユーザ名  ユーザー名

のように表示されます（ユーザー名の数は、そのコンピューターのユーザーの数によって異なるります）。


cd あなたのユーザー名
pwd

と入力してみましょう。すると

/Users/あなたのユーザー名

と表示され、元のディレクトリに戻ってきたことがわかります。
また、


cd Documents
pwd

と入力してみましょう。すると

/Users/あなたのユーザー名/Documents

と表示されます。つまり、Documents（書類）ディレクトリに移動しました。 ここで


ls

と入力すると、Documents ディレクトリの中のファイルやディレクトリを見ることができます。
ファイルやディレクトリが存在しない場合は、何も表示されません。

今度は、次のように入力してみましょう。 なお、~（チルダ）は Shift を押しながら、キーボード右上にある、ひらがなの「へ」が書いてあるキーを押すと入力できます。


cd ~
pwd

pwd の結果は、下記のように表示されるはずです。

/Users/あなたのユーザー名/

元に戻ってきました。 このコマンドで使った ~ （チルダ) は、ホームディレクトリ を表します。

ホームディレクトリは、このログインしているユーザーが、自由にファイルやフォルダを利用して良い領域です。
Mac では「ユーザフォルダ」ともいいます。

今は あなたのユーザ名 というユーザーでログインしているので、ホームディレクトリは /Users/あなたのユーザー名 であり、cd ~ は、 cd /Users/あなたのユーザー名 と同じ意味になります。

ディレクトリを移動していて、今どこにいるか分からなくなって困ったときなどは、cd ~ と入力して、一度ホームディレクトリに戻ってきましょう。

このように、ディレクトリを移動するときに、cd コマンドを使用しましょう。

mkdir コマンド

mkdir は、MaKe DIRectory（メイク・ディレクトリ）の略で、「ディレクトリを作成せよ」という意味のコマンドです。mkdir の後にスペースをあけて、作成するディレクトリ名を記述します。

mkdir のイメージ 

mkdir directory-test
ls

と入力すると

directory-test

と表示されるのではないでしょうか。 directory-test という名前のディレクトリが作成できました。

新しいディレクトリを作成したいとき、mkdir コマンドを使用しましょう。

rm コマンド

rm は、ReMove（リムーブ）の略で、「（ファイルやディレクトリを）削除せよ」という意味のコマンドです。
指定したファイルやディレクトリを削除します。rm の後にスペースをあけて、削除するファイル名またはディレクトリ名を記述します。

注意！ Finder における削除とは異なり、コマンドラインの削除では ゴミ箱への移動は行われず完全に削除されてしまいます。復元はできません。
rm コマンドを使うときは、本当に削除してよいファイルやディレクトリなのか、十分確認してください。
rm のイメージ 

cd directory-test
touch test.txt
ls

と入力してみましょう。

test.txt

と表示されるのではないでしょうか。

ここでは、まず test ディレクトリ内に移動し、touch というコマンドで test.txt というファイルを作成しました。touch はファイルを作成できるコマンドですが、ここで詳しくは触れません。
以上のコマンドで test.txt という名前のファイルが作成されました。

続いて、


rm test.txt

と入力してから


ls

と入力します。

すると、先ほどはあった test.txt がなくなりました。
rm コマンドでファイルを削除することができたことがわかります。

次は、ディレクトリの削除もしてみましょう。


cd ..
ls

で 1 つ上のディレクトリに戻り、その中身を確認します。 その中に、directory-test というディレクトリがあるはずです。

これは先ほど作ったディレクトリなので、rm コマンドで削除してみましょう。

注意！ 繰り返しになりますが rm コマンドでの削除の復元はできないので、間違えないよう十分注意してください。

次のように入力してください。


rm directory-test
  
rm: cannot remove ‘a’: Is a directory

という英語のエラーメッセージが表示されるはずです。

これは日本語に直すと「rm: ‘a’ はディレクトリです。削除できません」という意味です。

実は、 ディレクトリを rm コマンドで消す際には -r というオプションを付ける必要があります。


rm -r directory-test

上記のコマンドで削除が行えました。
確認してみましょう。


ls

ファイルとディレクトリの一覧から directory-test がなくなっているはずです。

ファイルやディレクトリを削除したいとき、rm コマンドを使用しましょう。しかし、復元はできないので十分注意しましょう。

恐怖の Unix コマンド sudo rm -rf /*

先ほどから再三注意しているとおり、rm での削除は、一度やってしまうと取り戻すことができません。
macOS でのコマンドの多くは Linux という OS でも使うことができ、両方を指して Unix コマンドといいます。
Unix コマンドの中で、恐怖のコマンド や 魔のコマンド として知られるのが sudo rm -rf /* です。


sudo は管理者権限でコマンドを実行するための記述です。

すでに学んだとおり、/ はルートディレクトリを表します。-rf は -r と -f を合わせて書いたオプションです。 -r はディレクトリを削除するためのオプション、-f は警告メッセージを出さないで強制的に実行するオプションです。そして、* はそのディレクトリの中の全てのファイルを表しています。


これを実行すると、何が起こるでしょうか？
全てのファイルやディレクトリは、どれもルートディレクトリの中にあります。
その中には、コンピュータを操作するための重要なファイルもあります。
これらのファイルがなくなると、コンピューターはほぼ使えなくなってしまします。 それが、たった 1 行のコマンド rm -rf /* で消えてしまうのです。


なお実際には、このコマンドを実行しても全てのファイルは消えないように OS の保護がかかっているかもしれません。しかし、ほとんどの重要なファイルはこれで消えてしまいます。また、同様の処理がされてしまう別の形式のコマンドもあります。


このようにルートディレクトリを指定するような極端な例ではなかったとしても、rm -r でディレクトリを削除すると、取り返しのつかないことになる可能性があります。rm コマンドの使用には十分注意しましょう。

まとめ

  • コマンドラインは、コンピューターに対してコマンドをキーボードから入力することで、コンピューターを操作する画面である。
  • コマンドラインは、コンピューターの高度な操作や設定をするとき、プログラミングをするときなどに使う。
  • Mac ではターミナルというソフトウェアを使う。
  • Finder でのファイル操作の代わりに、コマンドラインのコマンドを使ってファイル操作ができる。
